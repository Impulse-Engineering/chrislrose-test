<objective>
## Goal
Create migration scripts to export data from Supabase and import into D1, with data transformation (booleans, dates, FK ordering) and row count verification.

## Purpose
The new Hono app needs real data from the existing Supabase-backed site. Migration must handle type differences (Supabase booleans → D1 integers, date formats), FK ordering (categories before links), and collection link_ids format.

## Output
- scripts/export-supabase.ts — exports all tables from Supabase to JSON files
- scripts/migrate.ts — reads JSON, transforms data, imports to D1 via wrangler d1 execute
- Verification: row counts match between source and target
</objective>

<task type="auto">
  <name>Task 1: Create export and migration scripts</name>
  <files>scripts/export-supabase.ts, scripts/migrate.ts, scripts/tsconfig.json</files>
  <action>
    **1a. scripts/tsconfig.json:**
    Create a minimal tsconfig for the scripts directory:
    - target: ES2022, module: NodeNext, moduleResolution: NodeNext
    - strict: true, outDir: dist
    - Include scripts/*.ts

    **1b. scripts/export-supabase.ts:**
    Export script that connects to Supabase via REST API (not SDK — keep it simple):
    - Read SUPABASE_URL and SUPABASE_ANON_KEY from environment (or .env)
    - For each table (links, categories, collections, gear_hardware, gear_software, gear_hobbies, gear_projects, gear_podcasts, site_content):
      - GET all rows via Supabase REST: `${url}/rest/v1/${table}?select=*`
      - Write to scripts/data/${table}.json
    - Report row counts per table
    - Skip admin_users and sessions (those are created fresh in the new app)

    **1c. scripts/migrate.ts:**
    Import script that reads exported JSON and generates D1-compatible SQL:
    - Read JSON files from scripts/data/
    - Transform data:
      - links: `read` boolean → integer (true→1, false→0), `private` boolean → integer
      - links: generate TEXT id if missing (Supabase might use UUID — keep as-is if TEXT)
      - collections: `link_ids` array → JSON string if not already
      - Preserve null values
    - Generate INSERT statements in FK order:
      1. categories (no deps)
      2. site_content (no deps)
      3. gear_* tables (no deps)
      4. links (depends on categories)
      5. collections (no deps but logically after links)
    - Output: scripts/data/migrate.sql file with all INSERT statements
    - Also support direct execution via: `wrangler d1 execute DB --file=scripts/data/migrate.sql`
    - Report row counts and verification summary

    **Note:** These scripts run with Node.js (tsx or ts-node), NOT in Workers. They're development tools.
    Avoid: Using Supabase SDK — use plain fetch for simplicity. Don't modify the D1 schema.
  </action>
  <verify>npx tsc -p scripts/tsconfig.json --noEmit exits 0 (or tsx --check if available). Scripts exist and have correct structure.</verify>
