  <action>
    **1a. Modal HTML in pages.tsx:**
    Add the link modal HTML to the reading-list page route, before the closing `</Layout>` tag (after the admin FAB button). Use the same structure as the original reading-list.html lines 244-304:
    - Outer: div.rl-modal#link-modal[hidden] with backdrop + inner
    - Close button, title ("Add Link"), hidden link-edit-id input
    - Form fields: URL input, og-fetch-status div, title input, description textarea, note textarea, status select (with same options as filter), category select (empty — populated by JS), tags input, star-picker div (5 star spans), private checkbox, save-status div, Save button
    - Use the exact same IDs as the original (link-modal, link-modal-backdrop, link-modal-close, link-modal-title, link-edit-id, link-url, og-fetch-status, link-title, link-description, link-note, link-status, link-category, link-tags, star-picker, link-private, link-save-status, link-modal-save)
    - DO NOT use innerHTML — build with JSX

    **1b. Metadata endpoint (src/routes/api/meta.ts):**
    Create GET /api/meta?url=... endpoint:
    - Validate url query param (return 400 if missing)
    - Use fetch() to GET the URL with a browser-like User-Agent header
    - Parse the HTML response text for: og:title, og:description, og:image meta tags (also fallback to <title> and meta name="description")
    - Extract domain from URL, build favicon URL: `https://www.google.com/s2/favicons?domain=${domain}&sz=64`
    - Return JSON: { title, description, image, favicon, domain }
    - Handle fetch errors gracefully (return partial data or empty fields)
    - YouTube special case: if domain is youtube.com/youtu.be, use oEmbed API (`https://www.youtube.com/oembed?url=...&format=json`) for reliable title + use `https://img.youtube.com/vi/${videoId}/maxresdefault.jpg` for image
    - DO NOT require auth — metadata fetch is used during admin add flow but the endpoint itself is low-risk

    **1c. Register route in src/index.ts:**
    Import metaRoutes and mount at /api/meta

    Avoid: Using any npm packages for HTML parsing — use simple regex/string matching for OG tags (keeps bundle small on Workers).
  </action>
  <verify>npx tsc --noEmit exits 0. Modal HTML visible in page source at /reading-list.</verify>
