Create the D1 database schema, typed query helpers, PBKDF2 password library, cookie-based session management, auth middleware, and auth API routes (login/logout/session check).

## Purpose
All admin functionality (reading list CRUD, gear management, content editing) depends on authentication. This phase establishes the data layer and auth system that every subsequent phase builds on.

<task type="auto">
  <name>Task 2: Auth library, middleware, and API routes</name>
  <files>src/lib/password.ts, src/lib/session.ts, src/middleware/auth.ts, src/routes/api/auth.ts, src/index.ts</files>
  <action>
    1. Create src/lib/password.ts:
       - hashPassword(password: string): Promise<string> — PBKDF2-SHA256, 100000 iterations, random 16-byte salt
       - verifyPassword(password: string, hash: string): Promise<boolean> — extract salt from hash, re-derive, compare
       - Hash format: base64(salt):base64(derived_key) — simple, parseable
       - Use Web Crypto API (crypto.subtle.importKey, crypto.subtle.deriveBits)
       - Workers caps PBKDF2 at 100,000 iterations (design spec confirms this is acceptable)

    2. Create src/lib/session.ts:
       - createSession(db: D1Database, adminId: number): Promise<string> — generate crypto.randomUUID() token, insert into sessions with 7-day expiry, return token
       - validateSession(db: D1Database, token: string): Promise<Session | null> — lookup token, check expiry, return session or null
       - destroySession(db: D1Database, token: string): Promise<void> — delete from sessions
       - setSessionCookie(token: string): string — returns Set-Cookie header value: `session={token}; HttpOnly; Secure; SameSite=Lax; Path=/; Max-Age=604800`
       - clearSessionCookie(): string — returns Set-Cookie clearing the cookie

    3. Create src/middleware/auth.ts:
       - Hono middleware that reads `session` cookie from request
       - Calls validateSession to check token
       - If valid: set admin info on context (c.set('admin', session)) and call next()
       - If invalid/missing: return 401 JSON response
       - Export as `requireAuth` middleware

    4. Create src/routes/api/auth.ts:
       - POST /login — parse JSON body {email, password}, lookup admin by email, verify password, create session, set cookie, return 200
       - POST /logout — read session cookie, destroy session, clear cookie, return 200
       - GET /session — read session cookie, validate, return {authenticated: true/false, email}
       - All routes on a new Hono sub-app

    5. Update src/index.ts:
       - Import auth routes
       - Mount: app.route('/api/auth', authRoutes)
       - Create protected route group: app.use('/api/*', requireAuth) AFTER /api/auth routes (login must be unprotected)
       - Actually: mount auth routes first, then apply middleware to other /api/* routes
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - `wrangler dev` starts without errors
    - Manual test: POST /api/auth/session returns {authenticated: false}
  </verify>
  <done>AC-2, AC-3, AC-4, AC-5 satisfied: password lib works, login flow works, middleware protects routes, TypeScript compiles</done>
</task>
