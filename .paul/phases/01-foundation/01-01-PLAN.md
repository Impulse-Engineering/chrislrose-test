<task type="auto">
  <name>Task 1: Initialize project and configuration</name>
  <files>package.json, tsconfig.json, wrangler.toml, .dev.vars, src/types.ts</files>
  <action>
    1. Create package.json with:
       - Dependencies: hono
       - DevDependencies: @cloudflare/workers-types, wrangler, typescript
       - Scripts: "dev": "wrangler dev", "deploy": "wrangler deploy", "typecheck": "tsc --noEmit"
    2. Create tsconfig.json:
       - target: ESNext, module: ESNext, moduleResolution: bundler
       - jsx: "react-jsx", jsxImportSource: "hono/jsx"
       - types: ["@cloudflare/workers-types"]
       - strict: true, outDir: dist, rootDir: src
    3. Create wrangler.toml per design spec:
       - name: "chrislrose-dev"
       - compatibility_date: "2026-03-01"
       - D1 binding (database_id placeholder)
       - [assets] directory = "./public", binding = "ASSETS"
       - [vars] SITE_URL, ASSET_VERSION
    4. Create .dev.vars with placeholder comments (no real secrets)
    5. Create src/types.ts with:
       - Env type (DB: D1Database, ASSETS: Fetcher, SITE_URL: string, ASSET_VERSION: string)
       - Data model interfaces: Link, Category, Collection, GearItem, SiteContent, AdminUser, Session
       - Match fields exactly from design spec schema
    6. Run npm install
  </action>
  <verify>
    - `npm install` completes without errors
    - `npx tsc --noEmit` passes (may need dummy index.ts first)
  </verify>
  <done>AC-3 satisfied: TypeScript compiles with zero errors</done>
</task>

<task type="auto">
  <name>Task 2: Create Hono app, Layout/Nav components, homepage route, and migrate CSS</name>
  <files>src/index.ts, src/components/Layout.tsx, src/components/Nav.tsx, src/routes/pages.tsx, public/styles.css</files>
  <action>
    1. Create src/index.ts:
       - Import Hono, mount page routes
       - Typed with Env from types.ts: `new Hono<{ Bindings: Env }>()`
       - Mount pages routes at root
       - Export default app
    2. Create src/components/Layout.tsx:
       - Shared HTML shell: doctype, head (meta charset, viewport, OG tags, no-cache meta, Inter font link, styles.css with ?v= from env), body, footer
       - Accept props: title, description, children, env (for ASSET_VERSION and SITE_URL)
       - OG image: /og-image.png
    3. Create src/components/Nav.tsx:
       - Sticky frosted-glass nav matching existing site
       - Links: Home, About, Contact, Uses, Reading List
       - Active state via current path prop
    4. Create src/routes/pages.tsx:
       - GET / — Homepage route using Layout + Nav
       - Render hero section matching existing index.html structure
       - Use Hono JSX (server-rendered)
    5. Copy existing styles.css to public/styles.css (exact copy, no modifications)
    6. Copy og-image.png and og-reading-list.png to public/
  </action>
  <verify>

    - `curl http://localhost:8787/` returns HTML with nav, hero, footer, and stylesheet link
  </verify>
