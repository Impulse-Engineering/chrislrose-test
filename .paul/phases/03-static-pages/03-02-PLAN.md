<objective>
## Goal
Create gear API endpoints serving data from D1, and build the Uses page with server-rendered shell + client-side JS that fetches from those APIs — replacing the Supabase-powered original.

## Purpose
The Uses page is the second half of Phase 3 (Static Pages). Unlike About/Contact which are pure static content, Uses is data-driven: gear items live in D1 tables and are fetched via API. This completes the "personal web presence" pages.

## Output
- `src/db/queries.ts` — Gear query functions added (6 tables)
- `src/routes/api/gear.ts` — Public GET endpoints for all gear categories + site_content
- `src/routes/pages.tsx` — `/uses` route with full page shell (tab bar, panels, "What I'm Doing Now")
- `src/index.ts` — Mount gear API routes
- `public/uses.js` — Client-side JS to fetch gear API and render cards (port of original uses.js)
</objective>

<task type="auto">
  <name>Task 1: Add gear queries and create gear API routes</name>
  <files>src/db/queries.ts, src/routes/api/gear.ts, src/index.ts</files>
  <action>
    1. Add gear query functions to `src/db/queries.ts`:
       - `getGearHardware(db)` → SELECT * FROM gear_hardware ORDER BY sort_order
       - `getGearSoftware(db)` → SELECT * FROM gear_software ORDER BY sort_order
       - `getGearHobbies(db)` → SELECT * FROM gear_hobbies ORDER BY sort_order
       - `getGearProjects(db)` → SELECT * FROM gear_projects ORDER BY sort_order
       - `getGearPodcasts(db)` → SELECT * FROM gear_podcasts ORDER BY sort_order
       - `getSiteContent(db, id)` → SELECT * FROM site_content WHERE id = ?
       - All return typed results using existing GearItem and SiteContent types

    2. Create `src/routes/api/gear.ts`:
       - Hono router with GET endpoints:
         - GET /hardware → returns JSON array
         - GET /software → returns JSON array
         - GET /hobbies → returns JSON array
         - GET /projects → returns JSON array
         - GET /podcasts → returns JSON array
         - GET /now → returns JSON object (site_content row for id="now") or 404
       - Each endpoint calls the corresponding query function and returns c.json()

    3. Update `src/index.ts`:
       - Import gearRoutes from `./routes/api/gear`
       - Mount BEFORE the requireAuth middleware line: `app.route('/api/gear', gearRoutes);`
       - This ensures gear endpoints are public (no auth required)
       - The existing `app.use('/api/*', requireAuth)` will still protect other /api/* routes
         because Hono middleware applies to routes registered AFTER it

    Avoid:
    - Do NOT add write/mutation endpoints (that's Phase 5 Admin System)
    - Do NOT modify the auth middleware or existing auth routes
    - Do NOT add pagination — gear lists are small
  </action>
  <verify>
    1. `npx tsc --noEmit` passes
    2. `npx wrangler dev` starts
    3. `curl http://localhost:8787/api/gear/hardware` returns JSON array (may be empty if no seed data)
    4. `curl http://localhost:8787/api/gear/now` returns JSON or 404
    5. Gear endpoints respond without auth cookie (public access)
  </verify>
  <done>AC-1, AC-5, AC-6 satisfied</done>
</task>
