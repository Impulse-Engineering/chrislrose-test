<objective>
## Goal
Build collections API, share page (/c/:id), curation mode, bookmarklet support, and share modal — enabling users to curate and share link bundles from the reading list.

## Purpose
Collections are the social/sharing feature of the reading list. Admin picks links, generates a share URL, recipients see a curated view with OG tags for rich previews in iMessage/social. Bookmarklet allows quick-saving links from any page.

## Output
- Collections CRUD API (POST to create, GET by ID)
- Collection share page at /c/:id with SSR OG tags + redirect
- Curation mode: selection → create share link → share modal
- Bookmarklet: quick-save URL via /reading-list?add=URL
- Share modal HTML in reading-list page
- Wire all Phase 6 TODO stubs in reading-list.js
</objective>

    **1b. Collections API (new file src/routes/api/collections.ts):**
    - POST /api/collections — protected (requireAuth), creates collection
      - Generate ID: 8-char random hex string
      - Accept body: { recipient, message, link_ids: string[] }
      - Store link_ids as JSON string
      - Return created collection with 201
    - GET /api/collections/:id — public, returns collection by ID
      - Parse link_ids from JSON string back to array
      - Return 404 if not found

    **1c. Register route in index.ts:**
    Import collectionRoutes and mount at /api/collections (before the catch-all auth middleware)

    **1d. Share page at /c/:id in pages.tsx:**
    Add GET /c/:id route that:
    - Fetches collection by ID from DB
    - If not found: return 404 page
    - If found: return HTML page with:
      - OG meta tags: og:title = "Chris's picks for {recipient}", og:description = "Articles handpicked from Chris Rose's reading list", og:image = /og-reading-list.png
      - Twitter Card meta tags
      - Inline JS that redirects to /reading-list?collection={id}
      - Simple loading message body ("Opening reading list…")
    - This is a standalone page — does NOT use Layout component (minimal HTML for fast OG crawling)

    **1e. Share modal HTML in pages.tsx reading-list route:**
    Add share modal HTML after the link modal, before admin FAB:
    - div.rl-modal#share-modal[hidden] with backdrop + inner
    - Title "Share this collection"
    - Description text
    - URL input (readonly) + Copy button
    - Done button

    Avoid: Using Layout for /c/:id — keep it minimal HTML for OG crawler performance.
